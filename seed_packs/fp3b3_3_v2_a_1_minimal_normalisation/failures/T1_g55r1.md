# 1. What was attempted.

I attempted the T1 retry landing for handle `g55r1` by implementing a new module

```text
pnp3/Magnification/AuditRoutes/ProvenanceFilterV2/V2_A_1_g55r1/CanonicalNormalise.lean
```

The attempted design stayed inside the V2-A.1 syntactic boundary:

- structural recursion on `FormulaCircuit n`;
- a local `normaliseNot` reducing only `not (const b)` and `not (not C)`;
- local `normaliseAnd` / `normaliseOr` constructors for complement,
  identity, and absorbing-constant rewrites;
- a `CanonicalShape` image invariant excluding the root forms
  `not (const _)` and `not (not _)`;
- semantic preservation and size non-increase lemmas for each local constructor;
- the required hard-minimum surface, including
  `canonicalNormalise_not_const_true`, `canonicalNormalise_not_const_false`,
  `canonicalNormalise_double_not`, the AND/OR complement reductions,
  the AND-true identities, `canonicalNormalise_doubleNotPad`, and
  `canonicalNormalise_seedGate`.

Two syntactic-complement encodings were tried:

1. a Boolean `isComplement` using syntactic equality between formula subtrees;
2. a Prop-valued `IsComplement a b := b = normaliseNot a ∨ a = normaliseNot b`
   to make the semantic preservation proof expose explicit equality witnesses.

Neither design used `argmin`, `ttFormula (eval C)`, `Classical.choose`,
semantic/truth-table normalisation, or meta-level `Decidable (eval _ _)`
normalisation.


of that negative information through a naive `cases hc` proof when the scrutinee
is definitionally `canonicalNormalise c`; the proof needs additional factored
helper lemmas that first establish identities for an abstract canonical term
`d`, then instantiate `d := canonicalNormalise c`.

# 3. Local vs global obstruction.

Local.

This is a proof-engineering obstruction, not evidence that the V2-A.1 structural
normalisation route is globally impossible.  The attempted definitions remained
structural and syntactic, and the mathematical shape still looks consistent:
constant negation is included; double negation is routed through the canonical
image invariant; and complement detection is syntactic rather than semantic.

The obstruction is local because the failed goals identify missing helper
lemmas/factoring, not a contradiction in the theorem surface.  In particular,
the next worker should avoid proving the public identity lemmas directly against
`canonicalNormalise c`.  Instead, first prove abstract canonical-term helper
lemmas such as:

```lean
normaliseAnd_const_true_canonical
    {d : FormulaCircuit n} : CanonicalShape d →
      normaliseAnd d (FormulaCircuit.const true) = d

normaliseAnd_true_const_canonical
normaliseOr_const_false_canonical
normaliseOr_false_const_canonical
```

and only then instantiate them with `canonicalNormalise_isCanonical c`.

# 4. What an integrator must know.

The most promising next integration plan is:

1. Keep the Prop-valued complement relation rather than a Boolean equality test;
   the semantic proof is clearer because the equality witness is explicit.
2. Factor **all** local canonical-output identity proofs before proving public
   `canonicalNormalise_*` lemmas.  Do not case-analyse
   `canonicalNormalise_isCanonical c` while the target still contains
   `canonicalNormalise c`; generalise to an abstract `d` first.
3. Strengthen the canonical invariant just enough to make local identities
   frictionless.  The current root-only invariant is enough conceptually, but a
   derived lemma package should expose:
   - `CanonicalShape.not_not_const_false` / `not_not_const_true` style facts;
   - `normaliseNot_normaliseNot_of_canonical`;
   - the four constant identity lemmas for AND/OR.
4. Avoid broad `split` proofs over `match a, b`; prove local constructor
   preservation by explicit cases over `a` and `b`, and discharge the remaining
   `if IsComplement ...` branches with named hypotheses.
5. The failed Lean file was deliberately not committed.  This report is the only
   artifact of the `g55r1` retry attempt.
